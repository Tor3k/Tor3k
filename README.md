# 📚 My Python Learning Journey

## 1. Calculadora
Mi primer proyecto en Python. Aquí aprendí la sintaxis básica del lenguaje: `print()`, `input()`, los tipos de datos (`int`, `float` y `str`), la creación de funciones con `def`, realizar operaciones matemáticas y construir un menú utilizando `while`, `if` y `elif`.

## 2. Cajero
En este proyecto comencé a trabajar con una lógica un poco más completa. Aprendí a validar las opciones de un menú, utilizar listas para almacenar información, agregar elementos con `append()`, recorrer listas con `for` y separar el programa en funciones para mantener el código más ordenado.

## 3. Agenda
Aquí reforcé el trabajo con listas y funciones, además de aprender a buscar, agregar, modificar y eliminar información dentro de una estructura de datos. También comencé a organizar mejor mis programas para que fueran más fáciles de leer y mantener.

## 4. Adivina el número
En este proyecto aprendí a utilizar el módulo `random`, crear distintos niveles de dificultad, trabajar con contadores y controlar el flujo del juego mediante bucles y condiciones.

## 5. Piedra, Papel o Tijera
Este proyecto me permitió practicar la toma de decisiones, el uso de `random`, la comparación de opciones y la organización de la lógica del juego de una forma más limpia.

## 6. Hangman
Este fue el primer proyecto que realmente encontré desafiante. La parte más difícil fue construir toda la lógica para validar correctamente el juego: comprobar cuándo la palabra estaba completamente descubierta, asegurar que el jugador ingresara solo una letra y evitar que pudiera repetir letras ya utilizadas.

A partir de este proyecto también decidí comenzar a escribir prácticamente todo el código en inglés (variables, funciones y gran parte del backend) para acostumbrarme a trabajar utilizando el idioma más común dentro del desarrollo de software.

## 7. TicTacToe
Este también fue un proyecto complejo. La idea de cómo resolver el juego siempre estuvo clara en mi cabeza, pero transformarla en código fue el verdadero desafío. Escribir desde cero toda la lógica para detectar cuándo un jugador había formado una línea ganadora fue la parte más difícil. Una vez escrita puedo leerla, entenderla y modificarla sin problemas, pero llegar a esa solución desde cero fue un gran ejercicio para desarrollar mi lógica de programación.

## 8. MineSweeper
Este proyecto marcó un cambio importante en mi forma de programar. Ya no estaba escribiendo toda la sintaxis desde cero; por primera vez comencé a reutilizar funciones, estructuras y soluciones desarrolladas en proyectos anteriores, adaptándolas y mejorándolas según las necesidades del nuevo programa. Aquí empecé a comprender la importancia de escribir código limpio, reutilizable y fácil de mantener, además de trabajar con matrices, separar la lógica del juego de la interfaz y organizar proyectos de una forma mucho más profesional.
